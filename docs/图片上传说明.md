## 📸 图片CDN加速方案

本项目使用 **jsDelivr CDN** 来加速图片加载，这是一个免费的、基于GitHub的CDN服务，在中国国内访问速度较快。

### 两种CDN方案

本项目支持两种CDN方案，**两种方案都可以正常工作**：

#### 方案一：使用专门的CDN仓库（推荐）⭐

- **CDN仓库**：`git@github.com:dadongshangu/CDN.git`
- **优点**：
  - 图片资源集中管理
  - 不影响主仓库大小
  - 更专业的资源管理方式
  - 便于复用图片资源
- **CDN链接格式**：
  ```
  https://cdn.jsdelivr.net/gh/dadongshangu/CDN@master/qiuxuan/图片文件名.png
  ```

#### 方案二：使用当前仓库

- **当前仓库**：`git@github.com:dadongshangu/qiuxuan.git`
- **优点**：
  - 图片和文档在同一仓库，便于管理
  - 不需要额外的仓库
- **CDN链接格式**：
  ```
  https://cdn.jsdelivr.net/gh/dadongshangu/qiuxuan@master/assets/images/图片文件名.png
  ```

**注意**：两种方案可以同时使用，互不影响。建议使用方案一（专门的CDN仓库）。

## 🚀 如何上传图片

### 方案一：上传到专门的CDN仓库（推荐）⭐

#### 步骤1：克隆或访问CDN仓库

```bash
# 如果还没有克隆CDN仓库
git clone git@github.com:dadongshangu/CDN.git
cd CDN

# 或者如果已经克隆，直接进入目录
cd CDN
```

#### 步骤2：准备图片文件

1. 在CDN仓库中创建 `qiuxuan/` 目录（如果不存在）
2. 将图片文件保存到 `qiuxuan/` 目录
3. 建议使用 `.png` 或 `.jpg` 格式
4. 文件名使用英文和下划线，例如：`physics_question_13.png`

#### 步骤3：提交图片到CDN仓库

```bash
# 在CDN仓库目录中
git add qiuxuan/你的图片文件名.png

# 提交更改
git commit -m "Add qiuxuan image: 你的图片文件名"

# 推送到GitHub
git push origin master
```

#### 步骤4：在文档中引用图片

使用以下格式的CDN链接：

```markdown
![图片描述](https://cdn.jsdelivr.net/gh/dadongshangu/CDN@master/qiuxuan/你的图片文件名.png)
```

**格式说明**：
- `dadongshangu` - GitHub用户名
- `CDN` - CDN仓库名
- `master` - 分支名（如果使用main分支，请改为main）
- `qiuxuan/你的图片文件名.png` - 图片在CDN仓库中的路径

---

### 方案二：上传到当前仓库

#### 步骤1：准备图片文件

1. 将图片文件保存到 `assets/images/` 目录
2. 建议使用 `.png` 或 `.jpg` 格式
3. 文件名使用英文和下划线，例如：`physics_question_13.png`

#### 步骤2：提交图片到GitHub

```bash
# 在qiuxuan仓库目录中
git add assets/images/你的图片文件名.png

# 提交更改
git commit -m "Add image: 你的图片文件名"

# 推送到GitHub
git push origin master
```

#### 步骤3：在文档中引用图片

使用以下格式的CDN链接：

```markdown
![图片描述](https://cdn.jsdelivr.net/gh/dadongshangu/qiuxuan@master/assets/images/你的图片文件名.png)
```

**格式说明**：
- `dadongshangu` - GitHub用户名
- `qiuxuan` - 仓库名
- `master` - 分支名（如果使用main分支，请改为main）
- `assets/images/你的图片文件名.png` - 图片在仓库中的路径

- ✅ **稳定**：基于GitHub，稳定可靠

## 🔍 验证图片是否可用

上传图片后，可以在浏览器中直接访问CDN链接来验证：

```
https://cdn.jsdelivr.net/gh/dadongshangu/qiuxuan@master/assets/images/你的图片文件名.png
```

如果能看到图片，说明上传成功！

## 📌 注意事项

1. **文件大小**：建议单张图片不超过5MB，以保证加载速度
2. **文件格式**：推荐使用 `.png`（适合图表）或 `.jpg`（适合照片）
3. **命名规范**：使用英文和下划线，避免中文和特殊字符
4. **提交时机**：图片文件需要先提交到GitHub，CDN才能访问

## 🛠️ 故障排查

如果图片无法显示：

1. **检查文件是否已提交**：
   ```bash
   git ls-files assets/images/
   ```

2. **检查GitHub仓库**：访问 `https://github.com/dadongshangu/qiuxuan/tree/master/assets/images` 查看文件是否存在

3. **清除CDN缓存**：jsDelivr会自动更新，但可能需要几分钟时间

4. **检查链接格式**：确保CDN链接中的路径、分支名正确
